  <action>
1. Create `apps/api/src/app.ts`:
   - Export a `createApp()` factory function that returns a new `OpenAPIHono` instance
   - Type the app with `Bindings` (DB: D1Database, AUTH0_DOMAIN: string, AUTH0_AUDIENCE: string, X_API_KEY: string) and `Variables` (jwtPayload: { sub: string })
   - Export the `Bindings` and `Variables` types for use in middleware and route files
   - Export an `AppType` alias for the full `OpenAPIHono<{ Bindings: Bindings; Variables: Variables }>` type

2. Create `apps/api/src/middleware/api-key.ts`:
   - Export `apiKeyMiddleware()` using `createMiddleware` from `hono/factory`
   - Extract `x-api-key` header from request
   - If missing, throw `HTTPException(401, { message: 'Missing API key' })`
   - Compare against `c.env.X_API_KEY` using `crypto.subtle.timingSafeEqual` with TextEncoder
   - If length mismatch or comparison fails, throw `HTTPException(401, { message: 'Invalid API key' })`
   - Call `next()` on success

3. Create `apps/api/src/middleware/auth.ts`:
   - Export `jwtAuth()` function that returns the Hono `jwk` middleware configured for Auth0
   - Use `jwk` from `hono/jwk` with:
     - `jwks_uri: (c) => \`https://${c.env.AUTH0_DOMAIN}/.well-known/jwks.json\``
     - Use available config options from hono/jwk docs (alg, iss, aud verification)
   - The middleware sets `jwtPayload` on the context (accessible via `c.get('jwtPayload')`)

4. Create `apps/api/src/middleware/error-handler.ts`:
   - Export `errorHandler` -- a Hono error handler that catches errors and returns `{ error: { code: string, message: string } }` with appropriate HTTP status
   - For `HTTPException`, use its status and message
   - For Zod validation errors (from @hono/zod-openapi), return 400 with field-level error details
   - For unknown errors, return 500 with generic message
   - Export `notFoundHandler` -- returns 404 with `{ error: { code: 'NOT_FOUND', message: 'Not found' } }`

5. Rewrite `apps/api/src/index.ts`:
   - Import `createApp` from `./app`
   - Create app instance
   - Apply middleware: `app.use('/api/*', apiKeyMiddleware())`
   - Apply JWT auth on `/api/*` routes (will be overridden per-route for public endpoints in Plan 02)
   - Register `onError` and `notFound` handlers
   - Set up OpenAPI doc endpoint: `app.doc31('/doc', { openapi: '3.1.0', info: { title: 'DT65 API', version: '1.0.0' } })`
   - Set up Scalar UI: `app.get('/scalar', apiReference({ url: '/doc' }))` using `@scalar/hono-api-reference`
   - Add a health check route at `GET /` returning `{ name: DT65_APP_NAME, status: 'ok' }` (no auth required)
   - Export default app
   - Remove the old Hono import and VarLock ENV usage

6. Add `AUTH0_DOMAIN` and `AUTH0_AUDIENCE` as vars in `apps/api/wrangler.jsonc`:
   - Add to the top-level config (not just env.production): `"vars": { "AUTH0_DOMAIN": "dev-placeholder.auth0.com", "AUTH0_AUDIENCE": "https://dev-placeholder" }`
   - These are non-sensitive public values (Auth0 tenant domain and API identifier)

7. Run `pnpm check` and fix any lint/typecheck/knip issues. Common issues to watch for:
   - VarLock `ENV` import removal may affect varlock-noop-patches -- verify vite.config.ts still works
   - Biome may flag unused imports
   - Knip may flag new middleware files if not imported
