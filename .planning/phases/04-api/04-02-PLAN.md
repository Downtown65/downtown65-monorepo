1. Create `apps/api/src/routes/events/events.schemas.ts`:
   - Import `z` from `@hono/zod-openapi` (NOT from `zod` directly -- required for `.openapi()` method)
   - Import `EVENT_TYPES` from `@dt65/shared`
   - Define schemas with `.openapi()` metadata for OpenAPI spec generation:

   **EventTypeSchema**: `z.enum(EVENT_TYPES)` with openapi example 'RUNNING'

   **CreateEventSchema**: z.object with:
     - `type`: EventTypeSchema (required)
     - `title`: z.string().min(1).max(200) (required)
     - `dateStart`: z.string().date() (required, ISO date YYYY-MM-DD)
     - `timeStart`: z.string().optional() (HH:MM format)
     - `location`: z.string().max(200).optional()
     - `subtitle`: z.string().max(200).optional()
     - `description`: z.string().optional()
     - `race`: z.boolean().default(false)
   Call `.openapi('CreateEvent')` on the object

   **UpdateEventSchema**: Same as CreateEventSchema but all fields optional (partial). Call `.openapi('UpdateEvent')`

   **EventSchema**: Full event response shape including id, creatorId, createdAt, updatedAt, participantCount. Call `.openapi('Event')`

   **EventSummarySchema**: For list responses -- id, title, dateStart, timeStart, eventType, location, race, participantCount, creatorId. Call `.openapi('EventSummary')`

   **EventDetailSchema**: Full event with participants array (each with userId, nickname, joinedAt). Call `.openapi('EventDetail')`

   **ErrorSchema**: `z.object({ error: z.object({ code: z.string(), message: z.string() }) })`. Call `.openapi('Error')`

   **IdParamSchema**: `z.object({ id: z.string().openapi({ param: { name: 'id', in: 'path' }, example: '01HQ...' }) })`

2. Create `apps/api/src/routes/events/events.routes.ts`:
   - Import `createRoute` from `@hono/zod-openapi`
   - Define OpenAPI route objects (NOT handlers, just route definitions):

   **createEventRoute**: POST `/api/events`, request body CreateEventSchema, responses: 201 EventSchema, 400 ErrorSchema
   **listEventsRoute**: GET `/api/events`, responses: 200 array of EventSummarySchema
   **getEventRoute**: GET `/api/events/{id}`, params IdParamSchema, responses: 200 EventDetailSchema, 404 ErrorSchema. Tag as public (no JWT needed).
   **updateEventRoute**: PUT `/api/events/{id}`, params IdParamSchema, request body UpdateEventSchema, responses: 200 EventSchema, 403 ErrorSchema, 404 ErrorSchema
   **deleteEventRoute**: DELETE `/api/events/{id}`, params IdParamSchema, responses: 204 (no content), 403 ErrorSchema, 404 ErrorSchema

   No pagination for list endpoint -- ~20 events/month for ~100 members is manageable without pagination.
   Filter list to upcoming events only (dateStart >= today).

3. Create `apps/api/src/services/event-service.ts`:
   - Import Drizzle `drizzle` from `drizzle-orm/d1`, schema from `../db/schema`, and operators (`eq`, `gte`, `asc`, `sql`, `count`) from `drizzle-orm`
   - All functions accept `db: D1Database` as first argument and create drizzle instance inside, OR accept a drizzle instance -- pick the pattern that works with Hono context

   **createEvent(db, data, creatorId)**: Insert into events table with `crypto.randomUUID()` as id, createdAt/updatedAt as ISO timestamps. Return the created event.

   **getEventById(db, id)**: Select event by id. Join with usersToEvents + users to get participant list with nicknames and joinedAt. Return null if not found.

   **updateEvent(db, id, data, requesterId)**: First check event exists and creatorId matches requesterId. If not owner, return `{ ok: false, error: 'FORBIDDEN' }`. Update event fields and updatedAt. Return updated event.

   **deleteEvent(db, id, requesterId)**: Same ownership check. Delete event (cascade deletes participants). Return `{ ok: false, error: 'FORBIDDEN' | 'NOT_FOUND' }` or `{ ok: true }`.

   **listUpcomingEvents(db)**: Select events where dateStart >= today (use ISO date string comparison, YYYY-MM-DD format sorts lexicographically). Order by dateStart ASC. Include participant count via subquery or LEFT JOIN + GROUP BY. Return EventSummary array.

   Use Result type pattern per CLAUDE.md: `{ ok: true, data } | { ok: false, error }` -- no throw/catch for business logic.

   Handle D1 boolean: the `race` column stores 0/1 integers. Convert to boolean in service layer responses.
