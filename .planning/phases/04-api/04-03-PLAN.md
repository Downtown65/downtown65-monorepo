  <action>
1. Create `apps/api/src/services/participant-service.ts`:
   - Import Drizzle schema and operators

   **joinEvent(db, eventId, userId)**:
     - First verify the event exists and is upcoming (dateStart >= today). If event not found, return `{ ok: false, error: 'NOT_FOUND' }`. If past event, return `{ ok: false, error: 'PAST_EVENT' }`.
     - Use INSERT OR IGNORE (SQLite upsert) to add the user-event row with `joinedAt` set to current ISO timestamp. This makes the operation idempotent -- re-joining silently succeeds.
     - Return `{ ok: true }`.

   **leaveEvent(db, eventId, userId)**:
     - First verify the event exists and is upcoming. Same checks as join.
     - DELETE the user-event row. If no row existed, the DELETE succeeds with 0 affected rows -- idempotent.
     - Return `{ ok: true }`.

   Use Result type pattern: `{ ok: true } | { ok: false, error: 'NOT_FOUND' | 'PAST_EVENT' }`.

2. Add participation schemas to `events.schemas.ts`:
   - **ParticipantSchema**: z.object with userId, nickname, joinedAt. Call `.openapi('Participant')`
   - Ensure EventDetailSchema includes participants array using ParticipantSchema

3. Add participation routes to `events.routes.ts`:
   - **joinEventRoute**: POST `/api/events/{id}/participants`, params IdParamSchema, responses: 200 (success message or participant list), 400 ErrorSchema (past event), 404 ErrorSchema
   - **leaveEventRoute**: DELETE `/api/events/{id}/participants`, params IdParamSchema, responses: 200 (success message), 400 ErrorSchema (past event), 404 ErrorSchema

4. Add participation handlers to `events.handlers.ts`:
   - **handleJoinEvent**: Get `id` from params, `sub` from JWT. Call `joinEvent(c.env.DB, id, sub)`. Map errors to HTTP status codes (404 for NOT_FOUND, 400 for PAST_EVENT). Return 200 on success.
   - **handleLeaveEvent**: Same pattern with `leaveEvent`.

5. Register participation routes in `events.index.ts`:
   - Add `app.openapi(joinEventRoute, handleJoinEvent)` and `app.openapi(leaveEventRoute, handleLeaveEvent)`
   - Both routes require JWT auth (not public)

6. Run `pnpm check` and `pnpm --filter @dt65/api build`.
