<task type="auto">
  <name>Task 1: Install Drizzle dependencies and create schema + config</name>
  <files>
    pnpm-workspace.yaml
    apps/api/package.json
    apps/api/src/db/schema.ts
    apps/api/drizzle.config.ts
    apps/api/tsconfig.json
  </files>
  <action>
1. Add drizzle-orm and drizzle-kit to pnpm-workspace.yaml catalog:
   - `drizzle-orm: 0.45.1`
   - `drizzle-kit: 0.31.10`

2. Add dependencies to apps/api/package.json:
   - `drizzle-orm` as dependency (use `catalog:`)
   - `drizzle-kit` as devDependency (use `catalog:`)

3. Run `pnpm install` from repo root.

4. Create `apps/api/src/db/schema.ts` with the Drizzle schema from 03-RESEARCH.md Pattern 1:
   - `users` table: id (text PK, Auth0 sub), nickname (text notNull), picture (text notNull), createdAt (text notNull), updatedAt (text notNull)
   - `events` table: id (text PK, ULID), title (text notNull), subtitle (text), eventType (text notNull), dateStart (text notNull), timeStart (text), location (text), description (text), race (integer notNull default 0), creatorId (text notNull references users.id), createdAt (text notNull), updatedAt (text notNull)
   - `events` indexes: events_date_start_idx, events_event_type_idx, events_creator_id_idx
   - `usersToEvents` join table: userId (text notNull references users.id), eventId (text notNull references events.id with onDelete cascade), composite primary key on (userId, eventId)

5. Create `apps/api/drizzle.config.ts`:
   ```typescript
   import { defineConfig } from 'drizzle-kit';
   export default defineConfig({
     out: './drizzle',
     schema: './src/db/schema.ts',
     dialect: 'sqlite',
   });
   ```

6. Update `apps/api/tsconfig.json` to ensure the new db directory is included (it should be by default if src/ is included).

Use import from `drizzle-orm/sqlite-core` for sqliteTable, text, integer, index, primaryKey. Follow Biome conventions: single quotes, semicolons, 2-space indent. Use `import type` where appropriate per `verbatimModuleSyntax`.
  </action>
  <verify>
    <automated>cd apps/api && npx drizzle-kit generate 2>&1 | head -20</automated>
    <manual>Check that drizzle/0000_*.sql was created with CREATE TABLE statements for users, events, users_to_events</manual>
  </verify>
  <done>Drizzle schema file exists with 3 tables, drizzle.config.ts points to it, drizzle-kit generate produces a SQL migration file in apps/api/drizzle/</done>
</task>

<task type="auto">
  <name>Task 2: Configure D1 binding and apply migration locally</name>
  <files>
    apps/api/wrangler.jsonc
    apps/api/.gitignore
  </files>
  <action>
1. Update `apps/api/wrangler.jsonc` to add D1 binding:
   ```jsonc
   "d1_databases": [
     {
       "binding": "DB",
       "database_name": "downtown65-db",
       "database_id": "local-dev-placeholder",
       "migrations_dir": "drizzle"
     }
   ]
   ```
   Use a placeholder database_id for now -- the real ID will be set when creating the remote D1 database (Phase 4 or production setup). Local development uses `--local` which ignores the ID.

2. Update `apps/api/.gitignore` to add:
   - `data/` (for DynamoDB/Auth0 JSON exports used by migration script in Plan 02)

3. Apply the schema migration to the local D1 database:
   ```bash
   cd apps/api && npx wrangler d1 migrations apply downtown65-db --local
   ```

4. Verify tables exist by running:
   ```bash
   cd apps/api && npx wrangler d1 execute downtown65-db --local --command "SELECT name FROM sqlite_master WHERE type='table' ORDER BY name;"
   ```
   Expected tables: _cf_KV (wrangler internal), d1_migrations, events, users, users_to_events.

5. Verify foreign keys and indexes by inspecting the generated SQL file content.

6. Run `pnpm check` from repo root to ensure linting, typecheck, and knip all pass. Fix any issues (e.g., add drizzle-kit to knip config if needed since it's a CLI tool not imported in source).
