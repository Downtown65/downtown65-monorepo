  <action>
Create `apps/api/scripts/seed-from-dynamo.ts` -- a TypeScript script run via `npx tsx`.

The script must:

1. **Read source files** from `apps/api/data/`:
   - 4 DynamoDB event JSON files (glob `data/events-dump-*.json` or read specific names)
   - 1 Auth0 user export (`data/auth0-users.json`)
   - Print error and exit if any file is missing

2. **Parse Auth0 users** with Zod validation:
   - Expected format per user: `{ Id: string, Nickname: string, Picture: string, "Created At": string, "Updated At": string }`
   - Build a `Map<string, User>` keyed by `Id` (Auth0 sub)
   - Print: "Parsed N users from Auth0 export"

3. **Parse and merge DynamoDB events**:
   - Each JSON file contains an array of DynamoDB items
   - Filter items where `PK` starts with `EVENT#` (some files may have other item types)
   - Extract event ID from PK: `EVENT#<ULID>` -> `<ULID>`
   - Deduplicate by event ID (ULID) across all 4 files -- keep the first occurrence
   - Validate each event with Zod schema covering expected fields:
     - `PK`, `SK`, `title`, `dateStart`, `type` (maps to eventType), `race` (boolean)
     - Optional: `subtitle`, `timeStart`, `location`, `description` (raw HTML)
     - `createdBy`: `{ id: string, nickname: string, picture: string }`
     - `participants`: Record/map of `{ [auth0Sub]: { nickname: string, picture: string } }` or similar
     - `_ct` (created timestamp), `_md` (modified timestamp)
   - Print: "Parsed N unique events from N total across 4 files (N duplicates removed)"

4. **Field mapping** -- DynamoDB to D1:
   - `PK` (EVENT#ULID) -> `id` (just the ULID part)
   - `title` -> `title`
   - `subtitle` -> `subtitle` (nullable)
   - `type` -> `event_type`
   - `dateStart` -> `date_start`
   - `timeStart` -> `time_start` (nullable)
   - `location` -> `location` (nullable)
   - `description` -> `description` (nullable, raw HTML preserved)
   - `race` -> `race` (boolean `true`/`false` -> integer `1`/`0`)
   - `createdBy.id` -> `creator_id` (must exist in users map, FAIL if not found)
   - `_ct` -> `created_at` (ISO timestamp string)
   - `_md` -> `updated_at` (ISO timestamp string)

5. **Resolve creator IDs**:
   - For each event, look up `createdBy.id` in the Auth0 users map
   - If NOT found: print error with event ID and createdBy info, then FAIL the migration (per user decision)
   - If found: use the Auth0 `Id` as `creator_id`

6. **Resolve participation**:
   - For each event, iterate over `participants` map
   - For each participant auth0 sub, check if it exists in the Auth0 users map
   - If NOT found: print warning "Skipping orphaned participant {sub} for event {eventId}" and skip
   - If found: add (userId, eventId) to participation records
   - Track total skipped count for summary

7. **Generate seed.sql**:
   - Start with `PRAGMA defer_foreign_keys = ON;` for safety
   - Generate INSERT statements in dependency order: users first, then events, then users_to_events
   - Batch INSERTs into groups of ~50 rows per statement to stay under 100 KB limit
   - Use proper SQL escaping: replace `'` with `''` in all text values, handle NULL correctly
   - Write to `apps/api/data/seed.sql`
   - Print: "Generated seed.sql with N user inserts, N event inserts, N participation inserts"

8. **Print validation summary** to console:
   - Total users: N
   - Total events: N
   - Total participation records: N
   - Orphaned participants skipped: N
   - Any warnings accumulated during processing

Use `node:fs` and `node:path` for file I/O. Use `zod` (already in catalog) for validation. Follow Biome conventions. Use `import type` for type-only imports. No `console.log` -- use `process.stdout.write` or a simple logger function (Biome enforces noConsole; alternatively, add a biome-ignore comment for this script since it's a CLI tool that intentionally outputs to console).

Important: The Zod schema for DynamoDB events should be lenient on optional fields but strict on required ones. Log the first event parsed for manual inspection of the format.
