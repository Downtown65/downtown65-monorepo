Note: Look up current latest versions when installing. The above are from research -- use `catalog:` protocol in package.json.

**2. Update apps/www/package.json:**
- Add dependencies: `astro`, `tailwindcss`, `@tailwindcss/vite` (all as `catalog:`)
- Add devDependencies: `@astrojs/check` (as `catalog:`)
- Remove `@dt65/shared` dependency (www is a standalone static site, no shared types needed)
- Update scripts:
  - `"dev": "astro dev"`
  - `"build": "astro build"`
  - `"preview": "astro preview"`
  - `"typecheck": "astro check"` (replaces `tsc --noEmit` since Astro needs its own checker for .astro files)
  - Keep: `test`, `lint`, `format`, `ci:lint`, `clean` as-is
- Keep `"type": "module"`

**3. Create apps/www/astro.config.mjs:**
```javascript
import { defineConfig } from 'astro/config';
import tailwindcss from '@tailwindcss/vite';

export default defineConfig({
  vite: {
    plugins: [tailwindcss()],
  },
});
```

**4. Update apps/www/wrangler.jsonc:**
- Remove `"main": "src/index.ts"` (no Worker entry point for static site)
- Remove `"compatibility_flags": ["nodejs_compat_v2"]` (not needed for static assets)
- Add `"assets": { "directory": "./dist" }`
- Keep: `$schema`, `name`, `compatibility_date`, `observability`, `env`

**5. Update apps/www/tsconfig.json:**
- Astro manages its own TypeScript config. Replace the current config with one that extends Astro's strictest preset while keeping monorepo compatibility.
- Remove `"references"` to packages/shared (www does not import shared)
- Remove `"composite": true"` from compilerOptions if inherited -- Astro's check does not use project references
- Set `"extends": "astro/tsconfigs/strictest"` as the base
- Add compilerOptions for path aliases: `"paths": { "@/*": ["./src/*"] }`
- Include `"src"` in include array

**6. Delete apps/www/src/index.ts** (old Worker entry point)

**7. Update root tsconfig.json:**
- Remove the `{ "path": "apps/www" }` reference since Astro uses `astro check` instead of `tsc --build`
- This prevents `tsc --build` from trying to typecheck .astro files

**8. Update knip.json for apps/www workspace:**
- Change entry from `"src/index.ts"` to `["src/pages/**/*.astro", "astro.config.mjs"]`
- Remove `@dt65/shared` from ignoreDependencies (dependency will be removed)
- Add `"astro"` plugin recognition: knip has built-in Astro support, so setting correct entry patterns should suffice

**9. Run `pnpm install` after catalog and package.json changes.**

**10. Run `pnpm check` to verify lint, knip, and sherif pass.** Fix any issues (expect knip config tuning may be needed for Astro conventions). Run `pnpm typecheck` to verify the root tsc --build still works without the www reference.

Add the `.grid-background` utility class using CSS linear-gradient for the subtle dark grid pattern:
```css
.grid-background {
  background-color: var(--color-surface);
  background-image:
    linear-gradient(var(--color-surface-grid) 1px, transparent 1px),
    linear-gradient(90deg, var(--color-surface-grid) 1px, transparent 1px);

}
```

```css
.sticker {
```
